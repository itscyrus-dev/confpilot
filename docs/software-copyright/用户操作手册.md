# ConfigPilot 用户操作手册

## 一、启动软件

开发环境中运行：

```bash
export CONFIGPILOT_GITHUB_CLIENT_ID="你的 GitHub OAuth Client ID"
npm install
npm run tauri:dev
```

软件启动后进入首页，显示 GitHub 授权状态、仓库状态、配置文件扫描结果和最近同步时间。

## 二、GitHub 授权

1. 打开“授权”页面。
2. 点击“开始授权”。
3. 软件显示 GitHub 验证码。
4. 点击“打开 GitHub 授权页”。
5. 在 GitHub 页面输入验证码并确认授权。
6. 回到软件点击“我已授权”。
7. 授权成功后，软件自动确认或创建私有仓库。

## 三、配置扫描

软件自动扫描以下路径：

- `~/.zshrc`
- `~/.zprofile`
- `~/.zshenv`
- `~/.config/zsh/`
- `~/.config/ghostty/config`
- `~/.config/ghostty/`

存在的配置会显示为“已发现”，不存在的路径显示为“未发现”。

## 四、手动同步

进入“同步”页面后，可执行：

- 立即备份：将本地配置同步到 GitHub 私有仓库。
- 从云端恢复：将仓库中的配置恢复到本地路径。
- 双向同步：第一版按备份流程执行，并在冲突时提示用户处理。

## 五、自动监听

首页提供“开启监听”和“停止监听”按钮。开启监听后，软件会监控配置文件变化，并在短时间防抖后自动备份。

## 六、冲突处理

当本地与远端同一配置均发生变化时，软件进入冲突状态。用户可在“冲突”页面选择：

- 使用本地版本
- 使用远端版本
- 打开文件手动处理

冲突期间软件不会自动覆盖本地配置文件。

## 七、清理缓存

设置页面提供“清理本地缓存”功能。该操作只清理本机同步工作区，不删除 GitHub 私有仓库。
